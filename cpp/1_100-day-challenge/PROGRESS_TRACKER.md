## 🎯 Quick Tips

### When Stuck:
1. Read the problem 3 times
2. Draw examples on paper
3. Think of brute force first
4. Look for patterns in examples
5. Search for similar problems

### Code Quality Checklist:
- [ ] Meaningful variable names
- [ ] Functions < 30 lines
- [ ] No magic numbers
- [ ] Proper const usage
- [ ] Comments for complex logic
- [ ] Edge cases handled
- [ ] Memory leaks checked
- [ ] Compiles with -Wall -Wextra

### Before Submitting:
- [ ] Test with sample inputs
- [ ] Test edge cases (empty, single element, large)
- [ ] Check time complexity
- [ ] Check space complexity
- [ ] Code formatted (clang-format)
- [ ] No compiler warnings
- [ ] Valgrind clean (no leaks)
